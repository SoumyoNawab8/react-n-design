# Recommended GitHub Topics for react-n-design

Adding the right topics to a GitHub repository makes it discoverable in the Explore feed, topic search, and related-repository suggestions. Below are 20 recommended topics for react-n-design, grouped by category.

---

## Framework

| Topic | Why it improves discoverability |
|-------|-------------------------------|
| `react` | The library is built for React; this is the single most-searched topic on GitHub for UI libraries. |
| `typescript` | Typed component APIs are a major selling point; many developers filter by this tag. |
| `nextjs` | Next.js is the dominant React meta-framework, and RSC support makes this library directly relevant. |

## Design

| Topic | Why it improves discoverability |
|-------|-------------------------------|
| `neumorphism` | The library’s signature visual style; a niche but highly engaged design community actively searches for this term. |
| `neomorphic-ui` | Complements `neumorphism` and captures the "UI kit" intent rather than just the aesthetic concept. |
| `design-system` | Catches teams and developers looking for cohesive, documented component collections. |
| `ui-components` | A broad, high-volume tag that surfaces the library in general component-library searches. |

## AI

| Topic | Why it improves discoverability |
|-------|-------------------------------|
| `ai-components` | A fast-growing topic as developers search for drop-in AI interface primitives. |
| `chatbot-ui` | Directly describes the `AIChat` component use-case and attracts conversational-UX builders. |
| `ai-chat` | Overlaps with LLM app development; surfaces the library in AI-tooling and chat-interface searches. |
| `llm-ui` | Narrower and trending; targets developers wiring front-ends to OpenAI, Anthropic, or local models. |

## Architecture

| Topic | Why it improves discoverability |
|-------|-------------------------------|
| `rsc` | Short-hand for React Server Components; increasingly searched as Next.js App Router adoption grows. |
| `react-server-components` | The canonical long-form tag; ensures the library appears in deep RSC ecosystem searches. |
| `zero-runtime-css` | Signals styled-components are not required on the server path; appeals to performance-conscious adopters. |

## Quality

| Topic | Why it improves discoverability |
|-------|-------------------------------|
| `accessible` | Accessibility is a top evaluation criterion for component libraries; this tag filters to inclusive tools. |
| `a11y` | The standard community abbreviation; used by screen-reader users, auditors, and enterprise buyers. |
| `wcag-2.2` | Shows the library targets the latest accessibility standard, differentiating it from older, un-audited kits. |
| `tested` | Signals reliability; teams filtering for production-ready libraries look for this assurance. |
